Welcome to the central hub for community-driven modding documentation, guides, and discoveries!

## Overview

This folder contains the **11ty static site generator** configuration and source content for the modding community documentation site hosted on GitHub Pages at `https://fealyx.github.io/tvs/`.

## Building Locally

### Prerequisites
- Node.js 18+ and npm

### Setup
```bash
cd content
npm install
```

### Development Mode
```bash
npm run dev
```
Opens `http://localhost:8080` with live reload enabled. Edit any file in `src/` and the site rebuilds instantly.

### Production Build
```bash
npm run build
```
Generates the final `_site/` folder ready for deployment.

### Full Site Build (CI-equivalent)

To replicate the complete CI build including tool documentation collection from `tools/*/docs/`:
```bash
# From the repo root вЂ” collects tool docs and builds the 11ty site
rush build:pages

# Collect schemas separately if needed (writes to .pages/schemas/)
node common/scripts/collect-schemas.js
```

## Content Categories

- **Guides**: Long-form guides covering specific topics
- **Tutorials**: Step-by-step walkthroughs with examples
- **Discoveries**: Community findings, engine quirks, file format documentation
- **Community**: Contributor profiles, project roadmap, community announcements
- **Tools**: Documentation for modding tools (auto-integrated from `tools/*/docs`)

See [CONTRIBUTING.md](CONTRIBUTING.md) for detailed content guidelines and frontmatter requirements.

## Adding Content

1. Create a Markdown file in the appropriate category folder under `src/`
2. Include required frontmatter (see [CONTRIBUTING.md](CONTRIBUTING.md))
3. Write your content in Markdown
4. Run `npm run dev` to preview
5. Submit a PR for community review

## Adding Tool Documentation

Tool documentation is automatically collected from `tools/*/docs/` folders during the GitHub Pages build workflow. To make your tool documentation appear:

1. Add a `docs/` folder to your tool's directory (e.g., `tools/mynewotool/docs/`)
2. Include `README.md` and other documentation files
3. The tool docs will auto-appear in the site at `/tools/mytool/` on the next build

## Scripts

- `npm run dev` - Start dev server with live reload
- `npm run build` - Build static site to `_site/`
- `npm run watch` - Watch for changes without serving
- `npm run test:links` - Validate all links in generated site

## Publishing

The site is automatically built and published to GitHub Pages whenever changes are pushed to `main`:

1. GitHub Actions runs `.github/workflows/pages.yml`
2. Builds 11ty site + collects tool docs
3. Deploys `_site/` to GitHub Pages

For more details, see the workflow file: `.github/workflows/pages.yml`
